# YouTube 자막 업로드 오류 수정 보고서

## 문제 분석

### 증상
- YouTube에서 자막 파일 업로드 시 "선택한 파일을 파싱할 수 없습니다" 오류 발생
- 모든 언어의 자막 파일에서 동일한 오류 발생

### 원인
SRT 파일의 타임코드 형식이 YouTube 표준과 맞지 않음:
- **잘못된 형식**: `00:00:00 000 --> 00:00:02 970` (공백 사용)
- **올바른 형식**: `00:00:00,000 --> 00:00:02,970` (쉼표 사용)

YouTube는 SRT 표준 형식인 `HH:MM:SS,mmm` 형식만 허용하며, 밀리초 구분자로 쉼표(`,`)를 요구합니다.

## 해결 방법

### 1. 새로운 모듈 생성
**파일**: `srt_format_fixer.go`
- `FixSRTTimecodeFormat()`: 타임코드 형식을 자동으로 수정
- `ValidateSRTFormat()`: SRT 파일의 형식 검증

### 2. 실시간 수정 적용
**수정된 파일**: `main.go`, `translator.go`
- 텍스트 정리 단계에서 타임코드 형식 자동 수정
- 모든 번역 출력에 형식 수정 적용
- 한국어 교정 및 10개 언어 번역 모두 적용

### 3. 기존 파일 수정 도구
**파일**: `fix_existing_srt.go`
- CLI 명령어로 기존 파일 일괄 수정 가능
- 사용법: `bachata-bot-fixed.exe --fix-srt <디렉토리>`

